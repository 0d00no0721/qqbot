## SimPiano 钢琴旋律编码规范 v1.2

### 一、整体结构

text

```
[速度] 音符序列
```



- 曲谱为一个纯文本行（或多行），用**空格**将每个“节拍单元”隔开。
- 曲中可以任意插入小节线 `|`，仅用于提高可读性，**解析时一律忽略**。
- 小节线数量不限，可出现在任何空格位置前后。
- 节拍单元包括：单音、延音休止、分组括号 `( )` 或速度标记。

------

### 二、音高表示

**基本音级**

- `1`～`7` 对应 C、D、E、F、G、A、B。
- **中央 C（C4）定为 `1`**。向右：`2`=D4，`3`=E4，`4`=F4，`5`=G4，`6`=A4，`7`=B4。

**八度调整**

- 降低八度：在数字前加 `*`，可叠加（`*1`=C3，`**1`=C2 等）。
- 升高八度：在数字后加 `*`，可叠加（`1*`=C5，`1**`=C6 等）。
- `*` 必须紧贴数字，中间不能有空格。

**升降号**

- 升半音：`#`，写在数字或 `*` 之后。
- 降半音：`!`，同样后置。
- 组合示例：
  - 低一个八度的升 do：`*1#`（C#3）
  - 高一个八度的降 mi：`3*!`（Eb5）
  - 升 fa 升高两个八度：`4**#`（F#6）
  - 降 la 低两个八度：`**6!`（Ab2）

> `1!`（降 do）等同于 `*7`，`3!` 等同于 `*2#` 等，依方便选用。

------

### 三、时值与休止

**单音时值**

- 不加 `~` → 1 拍
- 加 1 个 `~` → 2 拍
- 加 2 个 `~` → 3 拍
- 加 n 个 `~` → (n+1) 拍

**休止符**

- 基本休止符：`-` 表示 1 拍休止。
- 延长规则相同：`-~` = 2 拍，`-~~` = 3 拍，……。
- 也可直接重复：`---` = 3 拍，`----` = 4 拍。
- 两种写法完全等效。

------

### 四、一拍多音：分组括号 `( )`

括号内所有元素合起来恰好**一拍**。内部用份额制分配节奏：

- 每个音符或休止符默认占 **1 份**。
- 每加一个 `~`，份额额外 **+1**。
- 所有元素份额总和即为该拍等分的份数；每个元素占 `份额/总份额` 拍。

**示例**

| 写法         | 份额计算                           | 实际节奏                                                     |
| :----------- | :--------------------------------- | :----------------------------------------------------------- |
| `(123)`      | 1+1+1=3                            | 三连音，各 1/3 拍                                            |
| `(1234)`     | 1+1+1+1=4                          | 四个十六分音符，各 1/4 拍                                    |
| `(1~~2~~3~)` | (1+2)+(1+2)+(1+1)=3+3+2=8          | 节奏 3-3-2，3/8、3/8、2/8 拍                                 |
| `(1-2-)`     | 音1,休1,音1,休1，共4               | 各 1/4 拍：音、休、音、休                                    |
| `(1-~2-~12)` | 音1,休(1+1)=2,音1,休2,音1,音1，共8 | 1/8 拍 do，休 1/4 拍，1/8 拍 re，休 1/4 拍，最后两个 1/8 拍 do、re |
| `(1~)`       | 1+1=2                              | 一个音占满 1 拍（可简写为 `1`，但兼容）                      |

**括号内以 `~` 开头**：表示该括号与前面的一个音符（或休止符）共同占用（前音符原时长 + `~` 个数）拍，`~` 不产生新音。多出的 `~` 拍数在前音符与括号内子音符之间按比例分配。

时长计算规则：
- 该单元总时长 = 前音符原时长 + `~` 个数（拍）。
- `~` 多出的拍数按以下权重分配：
  - 前音符的权重 = `~` 个数
  - 每个子音符的权重 = 该音符的份额（数字本身为 1 份，每个 `~` 加 1 份）
- 总权重 = 前音符权重 + 各子音符权重之和
- 前音符获得的额外时长 = `~` 个数 × 前音符权重 / 总权重
- 前音符最终时长 = 前音符原时长 + 前音符获得的额外时长
- 各子音符获得的时长 = `~` 个数 × 该音符权重 / 总权重

示例：
- `1 (~2)`：`~` 个数 = 1，前音符原时长 = 1。前音符权重 = 1，`2` 权重 = 1，总权重 = 2。前音符额外时长 = 1×1/2 = 0.5，前音符最终时长 = 1 + 0.5 = 1.5 拍。`2` 时长 = 1×1/2 = 0.5 拍。
- `1 (~~2~3)`：`~` 个数 = 2，前音符原时长 = 1。前音符权重 = 2，`2~` 权重 = 2，`3` 权重 = 1，总权重 = 5。前音符额外时长 = 2×2/5 = 0.8，前音符最终时长 = 1.8 拍。`2~` 时长 = 2×2/5 = 0.8 拍，`3` 时长 = 2×1/5 = 0.4 拍。
- `1~ (~2)`：前音符原时长 = 2（因自带 `~`），`~` 个数 = 1。前音符权重 = 1，`2` 权重 = 1，总权重 = 2。前音符额外时长 = 1×1/2 = 0.5，前音符最终时长 = 2.5 拍。`2` 时长 = 1×1/2 = 0.5 拍。

**注意**：括号内以 `~` 开头时，括号前面必须有一个音符（或休止符），否则视为语法错误。

**限制**

- 括号内至少含一个音符或休止。
- 括号内禁止出现速度标记 `[ ]`。
- 括号不可嵌套。

------

### 五、延音与重复音

- **延音**：在同音后加 `~` 表示一次触键、持续多拍。
  例：`1~~~` = 4 拍 do（一次弹下，持续）。
- **重复弹奏**：多次写相同数字并空格隔开。
  例：`1 1 1 1` = 每拍一下 do（四次触键）。

两者音效不同，记谱严格区分。

------

### 六、分隔与可读性

- **建议始终用空格**把一拍单元隔开。
- 在不致混淆时可省略空格（但不推荐）。
- 可能混淆时必须加空格。例如 `1*2` 应写为 `1* 2` 或 `1 *2`。
- 八度前缀 `*` 与前一拍之间也应用空格：`1 **2`。

------

### 七、小节线（可选，忽略）

- 字符 `|` 可插入在任意两个节拍单元之间，表示小节分界。
- `|` 仅为视觉提示，**实际解析时直接跳过**，不影响节奏与音高。
- 例：`1 1 5 5 | 6 6 5~ | 4 4 3 3 | 2 2 1~`

------

### 八、速度标记 `[bpm]`

- 格式：`[数字]`，数字为正整数，代表每分钟拍数。

- 可出现在**开头**，也可在**任意一拍之后**（空格后）单独使用，实现中途变速。

- **严禁**出现在分组括号 `( )` 内部（一拍中间不可变速）。

- 例：

  text

  ```
  [72] 1 2 3 4~
  [60] 1~~ [90] 2~ [120] 3
  ```

  

------

### 九、完整谱例

**1. 小星星（正确节奏）**

text

```
[80] 1 1 5 5 | 6 6 5~ | 4 4 3 3 | 2 2 1~
```



音符：C C G G | A A G(2拍) | F F E E | D D C(2拍)

**2. 带三连音、变速与升降**

text

```
[90] 1 (123) 5~ [100] (1~~2~~3~) 1* 
```



- 第1拍：C4 1拍
- 第2拍：C D E 三连音
- 第3拍：G4 2拍
- 变速100
- 第4拍：C D E 按 3-3-2 节奏（总1拍）
- 第5拍：C5 1拍

**3. 休止与复杂节奏**

text

```
[72] 3~ 2 1~ 2~ 3 3~~~ - 2~ 2~ 3 2 1~~~
```



解析：E4(2拍) D4(1拍) C4(2拍) D4(2拍) E4(1拍) E4(4拍)，休止1拍，D4(2拍) D4(2拍) E4(1拍) D4(1拍) C4(4拍)

------

### 十、使用与扩展说明

本编码专为**单旋律、单音**设计，极度精简，适合快速记谱、打字分享，或作为程序解析的基础格式。
若将来需要添加和弦、力度、连音线等，可在此规范上扩展新的符号层级（如用 `<>` 包裹和弦音，或 `pp`/`ff` 等标记），当前版本保持最简单实用。
