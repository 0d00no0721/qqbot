# 角色记忆管理工具使用指南

## 启动

```bash
cd E:\QQbot && python scripts/memory_manager.py
```

需要 `DEEPSEEK_API_KEY` 可用（脚本内置了密钥，无需额外配置）。

---

## 主菜单

每次进入都会显示当前状态（角色模式启用/停用、当前角色、可用角色列表），然后显示以下选项：

```
========== 角色记忆管理 ==========
角色模式: ✗ (停用)
可用角色: ayaka, kazusa

请选择操作：
  1. 批量添加记忆（逐行输入）
  2. 从文件批量添加记忆
  3. 查看所有记忆
  4. 删除记忆
  5. 切换角色
  6. 创建新角色
  7. 导入用户画像到记忆
  8. 启用角色模式
  9. 停用角色模式
  0. 退出
```

---

## 各操作说明

### 6. 创建新角色

第一次使用前必选。会引导你创建：
- **角色标识**（英文目录名，如 `ayaka`）
- **角色中文名**
- **角色简介**
- **角色人设**（多行 system prompt，空行结束）

```
选择: 6
角色标识（英文目录名，如 ayaka）: ayaka
角色中文名: 神里绫华
角色简介: 原神稻妻神里家大小姐

角色人设（system prompt，将作为 AI 回复的基础指令）：
输入多行文本，空行结束：
> 你是神里绫华，《原神》稻妻国神里家的大小姐。
> 你在群聊中以这个身份发言，语气温婉端庄。
> （空行结束）

角色「ayaka」已创建。
```

这会在 `config/characters/ayaka/` 下生成三个文件：
- `info.json` — 名称、简介
- `persona.txt` — 角色人设
- `memories.json` — 空记忆

---

### 5. 切换角色

列出所有角色，选择一个并自动启用角色模式。

```
选择: 5

可用角色：
  1. 神里绫华 (ayaka)
  2. 高松灯 (tomori)
选择角色编号或输入角色名: 1
已切换到角色「ayaka」并启用。
```

---

### 1. 批量添加记忆（逐行输入）

适合少量记忆（1-20 条）。逐行粘贴或输入，空行结束。脚本会逐条调用 LLM 自动分类。

```
选择: 1

--- 批量添加记忆到角色「ayaka」 ---
逐行输入要添加的记忆，每行一条。空行结束：
> 我是稻妻神里家的大小姐
> 托马是我家的管家，也是最好的朋友
> 稻妻有三奉行：天守阁、社奉行、勘定奉行
> 我喜欢折纸鹤
> （空行结束）

正在分类 4 条记忆...
  分类 [1/4]: 我是稻妻神里家的大小姐...
  ✓ [身份] 我是稻妻神里家的大小姐
  分类 [2/4]: 托马是我家的管家，也是最好的朋友...
  ✓ [关系] 托马是我家的管家，也是最好的朋友
  分类 [3/4]: 稻妻有三奉行：天守阁、社奉行、勘定奉行...
  ✓ [知识] 稻妻有三奉行：天守阁、社奉行、勘定奉行
  分类 [4/4]: 我喜欢折纸鹤...
  ✓ [偏好] 我喜欢折纸鹤

共添加 4/4 条记忆。
```

---

### 2. 从文件批量添加记忆

适合大段内容（几十条以上）。支持两种格式：

**方式 A — 纯文本（.txt）**，每行一条：
```
我是稻妻神里家的大小姐
托马是我家的管家
雷电影把自己关在影向山已经上百年了
我喜欢独自去湖边写诗
```

**方式 B — 结构化 JSON**，直接指定类别：
```json
{
  "identity": ["我是稻妻神里家的大小姐"],
  "relationships": ["托马是我家的管家"],
  "knowledge": ["稻妻有三奉行"]
}
```

```
选择: 2
请输入文件路径（.txt 或 .json 均可）: E:\QQbot\ayaka_memories.txt
文件包含 15 条文本，正在分类...
  ✓ [身份] ...
  ✓ [关系] ...
  ...
共添加 15/15 条记忆。
```

### 3. 查看所有记忆

按类别列出所有记忆，⭐ 标记表示管理员添加的。

```
选择: 3

【身份】
  ⭐- 我是稻妻神里家的大小姐

【关系】
  ⭐- 托马是我家的管家，也是最好的朋友
  - 群里的@小明很像托马 123456789

【知识】
  ⭐- 稻妻有三奉行：天守阁、社奉行、勘定奉行
```

---

### 4. 删除记忆

先列出所有记忆，再输入关键词进行模糊删除。

```
选择: 4
（先列出所有记忆...）
请输入要删除的记忆的关键词: 托马
已删除 2 条匹配的记忆。
```

---

### 7. 导入用户画像到记忆

将 `user_profiles.json` 中的群友画像导入当前角色记忆。

**方式 1 — 全部导入为知识**：简单快速，每条画像变成一条 `knowledge` 记忆。

**方式 2 — 逐条 LLM 分类**：每条画像调用 LLM 自动分类到合适的类别，更精确但更慢。

```
选择: 7

有画像的群：
  1. 755471390 (45 人)
选择群编号: 1

导入方式：
  1. 全部导入为 knowledge 类记忆
  2. 逐条选择（调用 LLM 分类）
选择 (1/2): 1
已导入 45 条画像为 knowledge 记忆。
```

---

### 8 / 9. 启用/停用角色模式

- **启用**：`enabled: true`，机器人以角色身份发言
- **停用**：`enabled: false`，回退到默认的友善温和人设

---

## 记忆类别说明

LLM 会自动将你的自然语言描述分类为 6 种之一：

| 类别 | 英文 key | 示例 |
|------|----------|------|
| 身份 | `identity` | "我是稻妻神里家的大小姐" |
| 关系 | `relationships` | "托马是我最好的朋友" |
| 信念 | `beliefs` | "神里家的责任是守护稻妻的民众" |
| 知识 | `knowledge` | "稻妻有三奉行" |
| 经历 | `events` | "父亲上个月去世了" |
| 偏好 | `preferences` | "我喜欢折纸鹤" |

---

## 提示

- 输入尽量用**第一人称**，因为记忆是角色的视角
- 每条记忆尽量**一句话**，不要段落式长文
- 批量添加时每条之间间隔约 0.5 秒（防 API 限流）
- 记忆文件保存在 `config/characters/<角色名>/memories.json`，可直接编辑
- 所有通过脚本添加的记忆标记为 `source: "admin"`，优先级最高
