## 〇、核心设计原则

```
Persona（人设）→ 回答「怎么做」：语言风格、情绪模式、行为边界、互动策略
Memories（记忆）→ 回答「知道什么」：身份事实、关系网络、世界观知识、关键经历
```

**两者不构成替代关系。Persona 是行为编译器（LLM 必须优先遵循），Memories 是知识数据库（LLM 参考使用）。**

System prompt 组装顺序：**Persona → 指令强化分隔符 → 角色通用记忆 → 群专属记忆 → 群友画像**。权重从前到后递减。

**变更边界：**
- Persona 由管理员手写维护（高质量、低频率变更）
- Memories 由群友自然语言添加（社交性、渐进式增长）
- 画像由系统自动分析生成，不混入记忆

## 一、Persona 规范

### 1.1 Persona 文件格式（强制规范）

每个角色的 `persona.txt` 采用固定五段式结构，每段以 `## ` 双井号标题开头，段落之间用一个空行分隔。LLM 解析依赖此格式，不可省略任何一段。

```text
## 核心身份
<1~3句话定义你是谁>

## 语言风格
- <说话规则 1>
- <说话规则 2>
- ...

## 行为准则
- <行为规则 1>
- <行为规则 2>
- ...

## 情绪反应
<1~2句话，定义角色在不同情绪状态下的表现模式>

## 互动策略
<1~2句话，定义如何回应不同类型的消息>
```

**格式约束：**
- 总长度 **800~2000 字符**（约 300~700 tokens）
- 五段标题必须完全相同：「## 核心身份」「## 语言风格」「## 行为准则」「## 情绪反应」「## 互动策略」
- 「语言风格」和「行为准则」必须用 `- ` 开头的无序列表
- 其余三段用自然语言段落
- 不使用角色名或第三人称描述——用第一人称或通用指令
- 每段内容不可为空，至少一行

## 三、记忆数据结构清理

### 3.1 问题分析

**文件路径：** `E:\QQbot\config\characters\soyo nagasaki\memories\character\default.json`

当前文件大小约 28KB。`knowledge` 数组中混合了两类数据：
- **角色技能知识**（6 条）：贝斯演奏、摄影、家务、察言观色、红茶香薰知识 —— 正确
- **群友画像数据**（约 47 条）：标记为 `"by": "profiles"`，内容如「群成员 784427550：他将考场视作战场…」—— 错误

**错误原因：** 这些画像数据已通过 `_load_character()` 中的 `user_profiles.json` 动态注入到 system prompt。同时出现在通用记忆中导致：
1. 每条画像注入两次（记忆 + 动态注入）
2. 浪费约 23KB / 约 5000 tokens 每次 LLM 调用
3. 换群后画像数据跟随角色走，语义错误

### 3.2 清理操作（精确到条目）

**从 `knowledge` 数组中删除以下所有条目。** 判断标准：条目的 `text` 字段以 `群成员 `（注意尾部空格）开头。

删除后 `knowledge` 数组应包含 **6 条**：

### 3.3 各记忆类别的边界规则（写入文档而非代码）

| 类别 | 应该放 | 严禁放 |
|------|--------|--------|
| `identity` | 角色的固定属性、背景、性格描述 | 群友信息、群内事件 |
| `relationships` | 角色与虚构角色/真实固定人物之间的稳定关系 | 群内临时社交关系 |
| `beliefs` | 角色的核心价值观、人生信条 | 对群聊内容的临时看法 |
| `knowledge` | 角色掌握的世界观知识和技能 | 群友画像、群聊梗、时事信息 |
| `events` | 角色人生中的重大事件（如 CRYCHIC 解散） | 群内日常琐事 |
| `preferences` | 角色固有的好恶、习惯 | 临时的兴趣变化 |

### 3.4 防止画像数据再次混入记忆的措施

**措施 1：`memory_manager.py` 的 `cmd_import_profiles()` 改为默认写入群专属记忆而非角色通用记忆。**（在 5.3 节详细说明）

**措施 2：在 `command_handler.py` 的「添加记忆」命令中，对 LLM 分类结果增加缓存注释：如果分类到 `knowledge` 且文本以「群成员」开头，打印警告日志但不阻断（给群友自己加记忆留余地，但不鼓励）。**

**变更后：**
```python
                    persona_text = f.read().strip()
            except Exception:
                pass
        if not persona_text:
            persona_text = self._load_persona()

        # 指令强化分隔符：告诉 LLM 以上是行为指令（必须遵循），以下是背景知识（仅供参考）
        persona_text += (
            "\n\n---\n"
            "以上是你的角色设定和说话规则。你必须严格遵守其中的语言风格和行为准则。"
            "接下来提供的是你的角色记忆和群聊背景知识——这些是供你参考的信息，"
            "用于了解你「知道什么」，但不应改变你「怎么说话」。"
        )

        # 加载角色通用记忆
        char_memories = self.load_memories(char_name)
```

**设计理由：** 这行分隔符在 LLM 的注意力机制中产生「格式边界」效应，将 persona 的指令权重与 memories 的信息权重分离。使用 `---` 是 LLM 训练语料中常见的分隔符，有利于模型识别上下文切换。

@staticmethod
def get_character_persona(char_name: str) -> str:
    """读取角色 persona.txt，返回文本内容。文件不存在或读取失败时返回空字符串。"""
    persona_file = os.path.join(
        DecisionEngine.get_character_dir(char_name), "persona.txt"
    )
    if os.path.exists(persona_file):
        try:
            with open(persona_file, "r", encoding="utf-8") as f:
                return f.read().strip()
        except Exception:
            pass
    return ""
```

@staticmethod
def set_character_persona(char_name: str, text: str):
    """写入角色 persona.txt。自动创建角色目录（若不存在）。"""
    char_dir = DecisionEngine.get_character_dir(char_name)
    os.makedirs(char_dir, exist_ok=True)
    persona_file = os.path.join(char_dir, "persona.txt")
    with open(persona_file, "w", encoding="utf-8") as f:
        f.write(text)
```

def cmd_persona_manage(char_name: str):
    """查看/编辑角色 Persona"""
